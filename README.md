# hybrid-isoform-flow

hybrid-isoform-flow is a public, reproducible workflow for hybrid transcript isoform analysis. It wraps the historical IDP-fusion algorithm with modern Python 3 orchestration, Snakemake workflow rules, explicit GRCh38 configuration, and adapters that convert HISAT2 short-read alignments into the SAM and junction BED inputs expected by IDP.

The package starts from IsoSeq-derived long-read files generated by PacBio SMRT (Single Molecule, Real-Time) sequencing instruments, plus paired Illumina FASTQs. It runs short-read trimming and HISAT2 alignment, converts splice junction evidence for IDP, corrects long reads with FMLRC, aligns corrected long reads with GMAP, runs IDP-fusion in its original Python 2 runtime, converts GPD/genePred outputs to GTF, and prepares SUPPA PSI analysis.

## What Changed From IDP-fusion

hybrid-isoform-flow does not redesign the IDP-fusion algorithm. The upstream code is vendored under `third_party/IDP-fusion` and pinned in `third_party/IDP-fusion/UPSTREAM.md`.

This package adds:

- a Snakemake workflow from IsoSeq outputs and Illumina FASTQs to SUPPA-ready PSI files
- Python 3 CLI helpers under `idp-beta`
- GRCh38-first configuration for references, HISAT2, GMAP, IDP, and SUPPA
- HISAT2-to-IDP SAM and junction BED adapters
- generated per-sample/per-chromosome IDP `run.cfg` files
- synthetic public fixtures for tests and dry-runs
- tests for config parsing, CIGAR parsing, junction conversion, GPD normalization, and expression merging

Historical notes used mixed genome-build wording. The public workflow intentionally defaults to one consistent GRCh38 reference set to avoid cross-build splice-junction and annotation errors.

## Repository Layout

- `idp_beta/`: Python package, CLI, config readers, adapters, and workflow helpers
- `workflow/`: Snakemake entrypoint and rule modules
- `config/`: public templates for samples, references, IDP parameters, and workflow options
- `container/`: Python 2 Dockerfile for upstream IDP-fusion
- `third_party/IDP-fusion/`: pinned upstream IDP-fusion source and license
- `tests/`: unit tests for the public package
- `results_example/`: tiny static expected-output examples only

Unpublished manuscripts, figures, tables, private sample outputs, and historical scratch scripts are intentionally excluded.

## Inputs

Edit `config/samples.tsv`:

```tsv
sample_id	condition	isoseq_reads	short_read_1	short_read_2	external_tpm
Con1	control	data/isoseq/Con1.flnc.fa	data/illumina/Con1_R1.fastq.gz	data/illumina/Con1_R2.fastq.gz	data/expression/allSamples.tpm.tsv
```

`isoseq_reads` should be an IsoSeq/CCS/FLNC FASTA or BAM exported by PacBio SMRT (Single Molecule, Real-Time) sequencing software. Raw PacBio subread processing is not automated in v1.

Edit `config/references.yml` with a consistent GRCh38 reference set:

- genome FASTA
- chromosome sizes
- HISAT2 index prefix
- GMAP database path
- reference GPD and full refFlat/GPD annotation
- UCSC `genePredToGtf`
- SUPPA executable

Edit `config/workflow.yml` for chromosomes, threads, trimming options, FMLRC command template, HISAT2/GMAP options, container image, and SUPPA event classes.

## Usage

Install the wrapper:

```bash
python3 -m pip install -e .
```

Validate templates without requiring local data files:

```bash
idp-beta validate-config --no-check-files
```

Create a tiny artificial fixture:

```bash
idp-beta make-synthetic-fixture
```

Dry-run the workflow:

```bash
idp-beta dry-run
```

Run the workflow when Snakemake and external tools are installed:

```bash
idp-beta run --cores 10
```

Useful direct helpers:

```bash
idp-beta adapt-hisat2 --input work/hisat2/Con1.sam --out-sam results/idp_inputs/Con1.chr1.splicemap-like.sam --out-bed results/idp_inputs/Con1.chr1.splicemap-like.junctions.bed --chromosome chr1 --read-length 150
idp-beta make-idp-configs --sample-id Con1 --chromosome chr1
idp-beta run-idp --config-file work/idp_configs/Con1.chr1.run.cfg --container-image idp-beta-py2
idp-beta summarize
```

Build the IDP Python 2 runtime:

```bash
docker build -f container/Dockerfile.idp-py2 -t idp-beta-py2 .
```

## Outputs

Main outputs are written under `results/`:

- `results/idp_inputs/`: IDP-compatible SAM and junction BED files
- `results/fmlrc/`: corrected long-read FASTA files
- `results/long_read/gmap/`: corrected long-read PSL alignments
- `results/idp/`: per-sample/per-chromosome IDP outputs
- `results/gpd/`: merged IDP GPD files
- `results/gtf/`: GTF files converted from GPD/genePred
- `results/suppa/`: SUPPA event and PSI outputs
- `results/summary.tsv`: lightweight output summary

## Limitations

hybrid-isoform-flow v1 starts from IsoSeq-derived long-read files, not raw PacBio subreads. External enrichment, proteomics, GSEA, Cytoscape, Metascape, KOBAS, and target-specific validation analyses are outside the automated workflow and should be documented separately for each study.

FMLRC setup can vary by installation, especially the short-read MSBWT input. Configure `fmlrc_command` in `config/workflow.yml` for your environment. The template can reference `{input}` or `{long_read}`, `{output}`, `{short_read_1}`, `{short_read_2}`, `{trimmed_read_1}`, `{trimmed_read_2}`, `{short_read_msbwt}`, and `{fmlrc_options}`.
